# Flowtel User Testing Rollout Plan

This document is the current intentional rollout plan for Flowtel beta testing. The goal is to open only a few systems at a time so issues stay easy to localize and the testing experience stays calm, beautiful, and clean.

## Active now

### Phase 1 — Guest Flow + Profile Studio Beta

**Purpose**
- Test the guest Flowtel rhythm over time.
- Start collecting real user cycle/check-in data.
- Finalize the Profile Studio experience.
- Keep testing narrow so bugs are easier to isolate.

**Open in Phase 1**
- Guest check-in flow
- Suite / Lounge guest experience
- Mentor request flow
- Profile Studio drafting + submission
- Admin profile review queue

**Closed / deferred in Phase 1**
- Practitioner/admin Clock In
- Concierge Desk access for beta guests
- Full Flow FM curriculum rollout
- Broader moon-path progression testing

**Mentor policy**
- Only the founding mentor account should be visible to guests.
- In code, the mentor list is narrowed to admin/owner accounts in this phase.

**Reviewer policy**
- Submitted Priestess Profiles should be reviewed by the founding admin/owner account.

## Planned next

### Phase 2 — Concierge Desk + Practitioner Workflows

**Purpose**
- Open the practitioner/admin Clock In path.
- Test the Concierge Desk, witnessing, and practitioner workflows once the guest path feels stable.

**Expected focus**
- Clock In
- Concierge Desk queue
- Manager / practitioner workflow
- Turndown / witnessing continuity

## Later

### Phase 3 — Wider Flow FM Curriculum

**Purpose**
- Open more of the moon curriculum and supporting architecture after the simpler system is stable.

**Expected focus**
- Wider portal access
- Moon-path curriculum refinement
- Additional review flows
- Expanded practitioner ecosystem

## Guiding principle

Open less. Test deeply. Localize issues. Make each doorway feel intentional before the next one opens.
